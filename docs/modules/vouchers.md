# SahakariMS — Module: Vouchers

## Overview

The Voucher module is the heart of the accounting system. Every financial transaction in SahakariMS — deposits, withdrawals, EMI payments, interest postings — automatically generates a corresponding accounting voucher following double-entry bookkeeping.

---

## Voucher Types

| Type | Code | Purpose |
|------|------|---------|
| Journal Voucher | JV | General ledger entries, corrections |
| Receipt Voucher | RV | Cash received (deposits, EMIs, fees) |
| Payment Voucher | PV | Cash paid out (withdrawals, salaries, expenses) |
| Contra Voucher | CV | Cash to/from bank, vault to counter |
| Bank Debit Note | BD | Bank charges, direct debits |
| Bank Credit Note | BC | Bank interest, direct credits |

---

## Voucher Number Format

```
{TYPE}-{BRANCH_CODE}-{YEAR_BS}-{MONTH_BS}-{SEQUENCE}

Example:
  JV-KTM-2081-04-00234  — Journal Voucher, Kathmandu, Baisakh 2081, #234
  RV-PKR-2081-04-00089  — Receipt Voucher, Pokhara, Baisakh 2081, #89
```

## Auto-Generated Vouchers

| Event | Voucher Type | Debit | Credit |
|-------|-------------|-------|--------|
| Cash deposit | Receipt | Cash in Hand | Member Savings A/c |
| Cash withdrawal | Payment | Member Savings A/c | Cash in Hand |
| Cheque deposit | Journal | Cheque in Hand | Member Savings A/c |
| EMI payment (cash) | Receipt | Cash in Hand | Loan Receivable |
| Loan interest income | Journal | Loan Receivable | Interest Income |
| Loan disbursement (cash) | Payment | Loan Receivable | Cash in Hand |
| Interest posting (savings) | Journal | Interest Expense | Member Savings A/c |
| FD creation (cash) | Receipt | Cash in Hand | Fixed Deposit A/c |
| FD maturity (credit) | Payment | Fixed Deposit A/c | Member Savings A/c |
| Share purchase | Receipt | Cash in Hand | Share Capital |
| Salary payment | Payment | Salary Expense | Cash / Bank |
| Penalty income | Journal | Loan Receivable | Penalty Income |
| TDS deduction | Journal | Interest Expense | TDS Payable |

## Voucher Domain Logic

```csharp
// Domain/Entities/Accounting/Voucher.cs
public class Voucher : AggregateRoot
{
    private readonly List<VoucherEntry> _entries = new();
    public IReadOnlyList<VoucherEntry> Entries => _entries.AsReadOnly();

    public VoucherStatus Status { get; private set; }
    public string VoucherNumber { get; private set; }
    public VoucherType VoucherType { get; private set; }

    public decimal TotalDebit => _entries.Sum(e => e.DebitAmount);
    public decimal TotalCredit => _entries.Sum(e => e.CreditAmount);
    public bool IsBalanced => TotalDebit == TotalCredit;

    public static Voucher Create(
        VoucherType type,
        DateOnly voucherDate,
        string narration,
        Guid branchId,
        Guid createdBy,
        bool isAutoGenerated = false)
    {
        return new Voucher
        {
            VoucherType = type,
            VoucherDateAd = voucherDate,
            Narration = narration,
            BranchId = branchId,
            CreatedBy = createdBy,
            IsAutoGenerated = isAutoGenerated,
            Status = VoucherStatus.Draft
        };
    }

    public void AddEntry(Guid accountId, decimal debit, decimal credit, string narration)
    {
        if (Status == VoucherStatus.Posted)
            throw new DomainException("Cannot modify a posted voucher.");

        if (debit < 0 || credit < 0)
            throw new DomainException("Entry amounts must be non-negative.");

        if (debit > 0 && credit > 0)
            throw new DomainException("Entry cannot have both debit and credit amounts.");

        _entries.Add(new VoucherEntry(accountId, debit, credit, narration));
    }

    public void Post(Guid postedBy)
    {
        if (Status == VoucherStatus.Posted)
            throw new DomainException("Voucher is already posted.");

        if (_entries.Count < 2)
            throw new DomainException("Voucher must have at least 2 entries.");

        if (!IsBalanced)
            throw new DomainException(
                $"Voucher is not balanced. Debit: {TotalDebit:N2}, Credit: {TotalCredit:N2}");

        Status = VoucherStatus.Posted;
        PostedBy = postedBy;
        PostedAt = DateTime.UtcNow;

        AddDomainEvent(new VoucherPostedEvent(Id, VoucherType, TotalDebit));
    }

    public Voucher Reverse(string reason, DateOnly reversalDate, Guid reversedBy)
    {
        if (Status != VoucherStatus.Posted)
            throw new DomainException("Only posted vouchers can be reversed.");

        if (IsAutoGenerated)
            throw new DomainException(
                "Auto-generated vouchers cannot be reversed directly. " +
                "Reverse the source transaction instead.");

        // Create reversal voucher with swapped debit/credit
        var reversal = Create(VoucherType, reversalDate, $"Reversal of {VoucherNumber}: {reason}",
            BranchId, reversedBy, isAutoGenerated: false);

        foreach (var entry in _entries)
        {
            reversal.AddEntry(entry.AccountId,
                debit: entry.CreditAmount,   // Swap
                credit: entry.DebitAmount,   // Swap
                narration: $"Reversal: {entry.Narration}");
        }

        Status = VoucherStatus.Reversed;
        ReversedBy = reversedBy;
        ReversedAt = DateTime.UtcNow;

        return reversal;
    }
}
```
